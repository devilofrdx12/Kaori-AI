# Features Workspace Implementation Plan (Phase 6)

This document outlines the current errors, shortcomings, and the required updates to fully implement the "Features Part" (`Projects`, `Artifacts`, and `Code` tabs) scheduled for Phase 6 of the Kaori AI Godmode plan.

## User Review Required

> [!IMPORTANT]
> Please review this plan for the `Projects`, `Artifacts`, and `Code` features. These represent significant architectural additions to the app. Confirm if you want to proceed with building all three, or if we should focus on one specific feature (e.g., Code snippets) first.

## Open Questions

> [!WARNING]
> 1. Do you want `Projects` to act as folders that group specific chats and artifacts together, or are they entirely separate entities?
> 2. For the `Code` workspace, should we prioritize the Pyodide (Python WASM) code execution engine mentioned in the Godmode plan, or just start with a Snippet Vault?

## Current Errors & Limitations

1. **Sidebar State Disconnect:** 
   Currently, when navigating to the `Projects`, `Artifacts`, or `Code` tabs, the sidebar still displays the list of recent **Chat Threads**. It should dynamically change to show a list of Projects, Artifacts, or Code Snippets depending on the active tab.
2. **Missing Workspace UIs:** 
   In `chat-layout.tsx`, these tabs all route to a single placeholder screen ("This feature is scheduled for Phase 6..."). There are no dedicated components for these workspaces.
3. **Missing Database Schemas:** 
   The database currently only supports Users, Conversations, Messages, and some OAuth tokens. It completely lacks tables for `projects`, `artifacts`, and `snippets`.
4. **Missing Backend APIs:** 
   There are no API routes to perform CRUD (Create, Read, Update, Delete) operations for these new features.
5. **No AI Integration:** 
   Kaori's system prompt and toolsets currently do not have the ability to save snippets or generate artifacts.

## Proposed Changes

We will systematically build out the foundations for these features, starting with the UI state management, backend APIs, and finally the AI tool integration.
